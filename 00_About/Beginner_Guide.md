# Beginner Guide

## AI-Assisted Obsidian Workflows with GitHub Copilot

---

# 1. What Are We Building?

We are creating a shared knowledge system that combines:

- Obsidian
- GitHub Copilot
- AI agents
- semantic search
- project documentation
- architecture knowledge

This allows us to:

- find information faster
- preserve decisions
- reduce duplicate work
- improve onboarding
- connect project knowledge
- create institutional memory

---

# 2. What Is Obsidian?

Obsidian is:

```text
Markdown files + links + local knowledge graph
```

Important characteristics:

- files stay local
- plain markdown
- Git-friendly
- AI-friendly
- highly extensible

Think of it as:

```text
A personal and organizational knowledge operating system.
```

---

# 3. What Is MCP?

MCP means:

```text
Model Context Protocol
```

It allows AI agents to safely interact with tools and data.

In our case:

```text
AI Agent
   ↓
MCP Connector
   ↓
Obsidian Vault
```

This allows agents to:

- read notes
- search semantically
- create notes
- update notes
- navigate links
- maintain knowledge graphs

---

# 4. What Is Semantic Search?

Normal search:

```text
Find exact words.
```

Semantic search:

```text
Find related meaning.
```

Example:

Searching for:

```text
authentication architecture
```

may also find:

- OAuth notes
- login workflows
- identity systems
- security decisions

This is why semantic retrieval is powerful.

---

# 5. What Are AI Agents?

Agents are AI systems that can:

- search information
- reason about context
- use tools
- modify files
- execute workflows

Examples:

- summarize meetings
- create architecture docs
- generate backlinks
- organize notes
- improve documentation

---

# 6. Important Safety Principle

AI agents are assistants.

They are NOT:

- authoritative
- always correct
- autonomous decision-makers

Humans remain responsible.

Always review:

- large edits
- architecture changes
- taxonomy changes
- deletions

# 10. Beginner Best Practices

## DO

- write clear note titles
- use links
- keep notes small and focused
- review AI output
- use Git commits often

---

## DON'T

- let agents mass-edit notes
- create giant notes
- mix unrelated concepts
- ignore governance
